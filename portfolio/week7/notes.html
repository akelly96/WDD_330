Week 7 Notes and Questions

Notes

  • Use memoization (caches) to store variables that take a lot time to process/render
  • Use immediately invoked functions for temporary variables.
  • If you only need to run a function once, there is no need to create a reusable function.
  • Only use strict mode inside temporary functions.
  • You can redeine a function inside itself. Afther the function is called once, it will change.
  • Recursion is a function calling itself until a certain condition is met.
  • The wait() function works asynchronously and will deploy something in a specified amount of time.
  • Separate tasks into functions. One function should perform one task.
  • A promise is used to execute something after something else happens.
  • .then is used to tell the computer what to do after a promise is complete
  • .catch is used if the promise fails.
  • Writing async in front of a function causes that function to run asynchronously with others being called.
  • The fetch API is a promise.
  • The redirect method is used to redirect the user to a new URL.
  • .json() transforms JSON into a JavaScript object.
